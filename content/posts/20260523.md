## 今回の更新：v5モデルとG1/G2/G3対応

### モデルを v5 にアップデート

今回から使用モデルを **v5**（AUC=0.692）に切り替えました。v4との主な違いは「レースのグレード情報」を学習データに正しく組み込んだ点です。

### G1/G2/G3の学習データ問題を修正

実は v4 までのモデルには重大な欠陥がありました。

学習に使う過去データ（race_info）には、G1・G2・G3レースが**すべて「オープン」として登録されていた**のです。つまりモデルは「天皇賞もただのオープン競走」として学習していました。

一方で予想時の出馬表スクレイピングは、netkeiba の CSS クラス（`Icon_GradeType1/2/3`）を正しく読み取り G1/G2/G3 を識別していました。**学習時と予測時でグレード情報が食い違っていた**わけです。

v5 では過去成績データ（horse_results）のレース名から G1/G2/G3 を正規表現で抽出し、学習データのグレードスコアとして付与する処理を追加しました。

```
grade_score: G1=3, G2=2, G3=1, 非重賞=0
```

特徴量重要度では `race_grade_score` が 27 位（importance=14）に入り、v4 では 0 だった値が有意な特徴として認識されるようになりました。

### 印（◎○▲）表記への変更

今回から予想の表示を「1位/2位/3位」から競馬らしい印（◎本命・○対抗・▲単穴）に変更しました。数字が並ぶと馬番と区別しにくかったため、視覚的にわかりやすくなるよう改善しています。
